# ROADMAP

本路线图用于统一近期迭代节奏，按周推进，强调可交付与可回归。

## 目标窗口

- 时间范围：未来 4 周
- 总体目标：稳定新增站点能力、增强分析深度、提升前端可用性与公网稳定性

## Week 1: 稳定与规范化

1. 爬取层
- 完成现有站点脚本参数与日志规范统一
- 补齐基础失败重试与错误分类

2. 分析层
- 固化分析报告输出结构（analysis_report.json）版本号
- 增加数据质量日报脚本（缺失率、重复率）

3. 前端层
- 完成筛选联动回归测试清单
- 优化加载状态提示与异常兜底

4. 交付
- 更新 docs/DECISIONS.md
- 补充 README 常见故障排查

## Week 2: 新站点扩展能力

1. 爬取层
- 新增 1 个网站接入（优先 API 站点）
- 输出字段保持统一契约

2. 分析层
- 新站点并入统计与求职指导模块
- 增加站点维度对比图表

3. 前端层
- 新增站点筛选标签（可选）
- 验证新站点数据在卡片展示中无字段断裂

4. 交付
- 新增站点维护文档（抓取路径、分页机制、风险点）

## Week 3: 分析深化与服务化准备

1. 分析层
- 增加趋势分析（按天/周）
- 增加岗位赛道机会评分（规则可解释）

2. 前端层
- 支持分析模块与筛选联动（点击图表自动筛选）
- 新增排序能力（发布时间/相关度）

3. 基础设施
- 评估将分析接口服务化（HTTP API）
- 设计 API 契约草案

4. 交付
- docs/JOB_MARKET_ANALYSIS.md 增补解释

## Week 4: 公网可运营化

1. 部署与运维
- 完成 GitHub Pages 发布流程演练
- 增加发布后检查项（索引、分片、页面加载、筛选链路）

2. 质量保障
- 建立基础 CI 检查（脚本可运行、关键文件存在）
- 建立回归脚本（核心筛选路径、导出路径）

3. 可用性
- 补充免责声明、数据来源、更新时间一致性检查
- 优化移动端与低网速体验

4. 交付
- 形成月度复盘：完成项、问题项、下月优先级

## Backlog（持续池）

- 抽象通用爬虫基类
- 增量抓取机制（按 job_id/publish_time）
- 收藏与投递跟踪功能
- 公司画像页与岗位对比页
- 分析接口 HTTP 化

## 里程碑验收标准

1. 数据完整性
- 字段契约一致
- 新站点并入后分析与前端可直接消费

2. 功能完整性
- 筛选、搜索、统计、卡片展示稳定
- 新需求按模板提交并可验证

3. 发布稳定性
- main 推送后自动发布成功
- 公网页面可在首次加载后 3 秒内看到首批数据
